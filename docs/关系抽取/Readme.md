# 关系抽取

## conditional bert

*  降低二阶段关系抽取的计算量
* 二阶段关系抽取计算方案：先抽取主语，根据计算出来的主语标记输入语句，将标记后的语句重新输入到bert中预测与主语相关的宾语的关系
* 缺点：需要过两次bert,计算量消耗大。

## tplinker
* 引入了token pair linking的任务，,输出端通过一个K个NxN的矩阵，表示实体与关系，实体矩阵位置i,j表示从位置i开始到位置j结束的实体，解决了重叠实体与重叠关系的问题。
每个实体，或者每个关系做一个三分类任务，（取消了下三角，用三分类表示），每一个三分类任务需要引入一组注意力权重（768* ），在类别数目较多的情况下，参数量增加非常明显。
  
## 主要的工作
* 多模型投票法，获取表征性数据以及差异性数据
* 伪标签法，利用更多的数据
* 使用多标签分类任务替换多分类任务，解决某个或者某几个实体拟合效果不佳的问题【直接标注包含那个类型的数据，可以只标注一种类型，计算损失函数的时候，直接mask】
